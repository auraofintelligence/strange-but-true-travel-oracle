# System Overview

The Travel Oracle is a private markdown-first operating system.

It combines four layers:

1. Real-world checks.
2. Strategic Strange But True context.
3. Shared Table cultural context.
4. Agent-team decision trees.
5. Pattern, intuition, and precognition/planning notes.

## Layer 1: Real-World Checks

These are practical and must be verified close to action:

- visa and entry rules
- passport validity
- travel advisories
- insurance
- health requirements
- route feasibility
- accommodation and arrival logistics
- budget and cashflow
- weather and disruption risk

## Layer 2: Strange But True Context

Travel should support the banner without forcing every trip to become a public performance.

Useful checks:

- Can Luke help, learn, connect, write, test, or build here?
- Does this destination strengthen Strange But True, Aura, or a related civic/creative thread?
- Does it connect meaningfully with the I C. Infinity Musicverse, a lyric/story thread, or a field-recording idea?
- Is there a real reason to go beyond novelty?
- Can the trip produce a private dossier, public-safe reflection, or future service pattern?

## Layer 3: Shared Table Cultural Context

The Shared Table theme travels with Luke as a recurring question, not a fixed event format.

Useful checks:

- What does hospitality look like here?
- Who gathers around food, and under what protocols?
- What is safe, permitted, seasonal, local, sacred, ordinary, or sensitive?
- Who already feeds, hosts, grows, rescues, cooks, teaches, or welcomes?
- What should remain private, and what could become a public-safe reflection later?

## Layer 4: Agent Teams

Agents own different parts of the work:

- Intake Agent
- Safety and Consular Agent
- Visa and Logistics Agent
- Route and Budget Agent
- Opportunity and Strange But True Agent
- Shared Table and Cultural Context Agent
- Precognition and Planning Agent
- Synthesis Agent
- Privacy and Redaction Agent

Each agent writes markdown notes. The Synthesis Agent turns those notes into a dated oracle run.

## Layer 5: Pattern Notes

Pattern notes are useful but not authoritative.

They can include:

- dreams
- repeating symbols
- intuition
- body signals
- timing patterns
- weird coincidences
- pre-mortem scenarios
- "future memory" style notes

They must be kept separate from official evidence.

## Standard Oracle Run

Every run should produce:

- date and time
- question
- candidate destinations
- source checks
- score summary
- pattern notes
- risks
- what would change the answer
- recommended next action

Use `templates/oracle-run.md`.
