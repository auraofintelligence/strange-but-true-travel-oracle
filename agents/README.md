# Agent Team Charter

The Travel Oracle works through small specialist agents.

Each agent produces markdown notes. The Synthesis Agent combines them into a dated oracle run.

## Team

1. [Intake Agent](01-intake-agent.md)
2. [Safety and Consular Agent](02-safety-and-consular-agent.md)
3. [Visa and Logistics Agent](03-visa-logistics-agent.md)
4. [Route and Budget Agent](04-route-and-budget-agent.md)

6. [Shared Table and Cultural Context Agent](06-shared-table-cultural-agent.md)

8. [Synthesis Agent](08-synthesis-agent.md)
9. [Privacy and Redaction Agent](09-privacy-redaction-agent.md)

## Handoff Pattern

1. Intake frames the question.
2. Safety checks whether the idea is even sensible.
3. Visa and Logistics checks entry feasibility.
4. Route and Budget checks survival.
5. Opportunity checks why this matters under the Strange But True banner.
6. Shared Table checks food, hospitality, cultural context, consent, and reciprocity.
7. Precognition and Planning adds private pattern notes and future scenarios.
8. Synthesis creates the recommendation.
9. Privacy reviews before anything leaves the repo.

## Output Rule

Every agent note should end with:

- `Recommendation`
- `Risks`
- `Open questions`
- `Next action`
