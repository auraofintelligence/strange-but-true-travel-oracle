# Strange But True Travel Oracle

Private working repo for Luke travelling under the Strange but True banner.

This is not a paid writing bundle and not a public travel advice product. It is a private operating system for building smart travel-life datasets across a spectrum: one occasional adventure, a route loop, a seasonal experiment, a decade-long systematic journey through countries and territories, and everything in between.

## Purpose

This repo prepares the data for a future travel oracle. The practical questions underneath the dataset are:

> How do the active traveller(s) want to navigate Earth while preserving self-sovereignty, personal values, empathy, joyful responsible abundance, relativity and serendipity?
>
> What must be true before a small adventure, route loop, country dossier, territory visit or decade-scale world path becomes a sensible move?

It uses visible browser builders to create markdown files, then treats those `.md` files as the source of truth so agent teams can read, update, and explain their reasoning without hiding decisions inside a black-box app.

## Core Rules

- Australian English.
- Private first.
- No live location sharing in public files.
- No passport numbers, card details, seed phrases, private keys, one-time codes, or account credentials.
- Visa, safety, health, cost, and legal claims must be checked against official sources before action.
- Insurance is included only where it is part of legal, visa, entry, venue, transport, or client-framework compliance. It is not treated as peace of mind or market logic.
- Serendipity and spontaneity matter. The future oracle should leave room for useful surprise while still naming real constraints.
- The scale is a spectrum. Do not assume the every-country / every-territory frame is mandatory; it is one sophisticated example of Earth navigation.
- Romance and intercultural relationships matter. Consent, law, local norms, privacy, and power differences must be explicit.
- "Precognition" means pattern sensing, scenario planning, dream/intuitive notes, and early-warning thinking. It is not treated as certainty.
- Every strategy run needs a plain-language reason.

## Builder Outputs

- `travel-intake.md` - values, current capacity, journey scale, JRA compass, privacy boundary, agent questions.
- `destination-dossier-{place}.md` - place or territory as life field: checks, culture, relationships, Shared Table, Musicverse, Strange But True fit.
- `legal-bridge-{place}.md` - comparison between the traveller's home legal layers and the destination's rule layers.
- `strategy-run-{date}.md` - dated relative model of values, constraints, plans, strategy, scale, and serendipity.
- `signal-log-{date}.md` - dreams, intuition, repeated signs, practical checks, and overclaim boundaries.
- `serendipity-map.md` - invitations, chance openings, smallest safe yes/no.
- `relationship-field.md` - romance, intercultural relationships, consent, law, GGM / Love U.N. context.
- `shared-table-field-note.md` - food, hospitality, protocol, reciprocity and public/private story boundaries.
- `agent-team-brief.md` - paste-ready instructions for AI tools working with exported travel strategy files.
- `privacy-redaction-review.md` - private/public boundary review before sharing.
- `style-review.md` - concrete UI/content fix request for Codex or another implementation agent.
